## Supported Versions

The OWASP Nest project follows a rolling release model. Only the latest version on the `main` branch is actively supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in OWASP Nest, please report it responsibly.

### How to Report

<Warning>
Please use the [GitHub Security Advisory form](https://github.com/OWASP/Nest/security/advisories/new) to report a security vulnerability.

**Do not** create public GitHub issues for security-related reports.
</Warning>

### What to Include

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Any relevant logs or screenshots

## Response Timeline

We are committed to responding to security reports in a timely manner:

<Steps>
  <Step title="Initial Acknowledgment">
    Within **10 days** of receiving your report
  </Step>

  <Step title="Investigation and Validation">
    Within **30 days** depending on the issue severity
  </Step>

  <Step title="Fix and Coordinated Disclosure">
    **Within 90 days** after investigation and validation
  </Step>
</Steps>

Thank you for helping keep OWASP Nest and the community secure.
