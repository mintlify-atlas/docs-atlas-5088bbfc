## Finding Issues via the API

You can query issues programmatically using the REST API.

### List All Issues

```bash
GET /api/v0/issues/
```

**Query Parameters:**
- `organization` - Organization name (e.g., "OWASP")
- `repository` - Repository name (e.g., "Nest")
- `state` - Issue state: `open` or `closed`
- `ordering` - Sort order: `created_at`, `-created_at`, `updated_at`, `-updated_at`
- `page` - Page number for pagination
- `limit` - Items per page

<Tabs>
  <Tab title="cURL">
    ```bash
    # Get open issues from OWASP/Nest
    curl "https://nest.owasp.org/api/v0/issues/?organization=OWASP&repository=Nest&state=open"
    ```
  </Tab>
  
  <Tab title="Python">
    ```python
    import requests
    
    response = requests.get(
        "https://nest.owasp.org/api/v0/issues/",
        params={
            "organization": "OWASP",
            "repository": "Nest",
            "state": "open",
            "ordering": "-created_at"
        }
    )
    
    issues = response.json()
    ```
  </Tab>
  
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch(
      'https://nest.owasp.org/api/v0/issues/?' + 
      new URLSearchParams({
        organization: 'OWASP',
        repository: 'Nest',
        state: 'open',
        ordering: '-created_at'
      })
    );
    
    const issues = await response.json();
    ```
  </Tab>
</Tabs>

### Get Specific Issue

```bash
GET /api/v0/issues/{organization}/{repository}/{issue_number}
```

**Example:**

```bash
curl "https://nest.owasp.org/api/v0/issues/OWASP/Nest/1234"
```

**Response:**

## Filtering by Experience Level

<Steps>
  <Step title="Good First Issues">
    Perfect for newcomers to the project:
    
    ```bash
    # Search for beginner-friendly issues
    curl "https://nest.owasp.org/api/v0/issues/?organization=OWASP&state=open" \
      | jq '.results[] | select(.labels[]? | contains("good first issue"))'
    ```
    
    These issues typically:
    - Have clear descriptions
    - Require minimal context
    - Include guidance on implementation
    - Are smaller in scope
  </Step>
  
  <Step title="Intermediate Issues">
    For contributors with project knowledge:
    
    Look for issues labeled:
    - `enhancement` - Feature improvements
    - `bug` - Bug fixes requiring investigation
    - `refactoring` - Code quality improvements
  </Step>
  
  <Step title="Advanced Issues">
    For experienced contributors:
    
    Look for issues labeled:
    - `architecture` - System design changes
    - `performance` - Optimization work
    - `security` - Security enhancements
  </Step>
</Steps>

## Sorting and Ordering

Choose the right sort order for your workflow:

<Tabs>
  <Tab title="Newest First">
    ```bash
    curl "https://nest.owasp.org/api/v0/issues/?ordering=-created_at"
    ```
    
    Use this to find the latest reported issues.
  </Tab>
  
  <Tab title="Oldest First">
    ```bash
    curl "https://nest.owasp.org/api/v0/issues/?ordering=created_at"
    ```
    
    Use this to help with longstanding issues.
  </Tab>
  
  <Tab title="Recently Updated">
    ```bash
    curl "https://nest.owasp.org/api/v0/issues/?ordering=-updated_at"
    ```
    
    Use this to find active discussions.
  </Tab>
  
  <Tab title="Least Recently Updated">
    ```bash
    curl "https://nest.owasp.org/api/v0/issues/?ordering=updated_at"
    ```
    
    Use this to find stale issues that need attention.
  </Tab>
</Tabs>

## Working with Pagination

The API returns paginated results:

```json
{
  "count": 156,
  "next": "https://nest.owasp.org/api/v0/issues/?page=2",
  "previous": null,
  "results": [
    // ... issues ...
  ]
}
```

### Iterate Through All Pages

<Tabs>
  <Tab title="Python">
    ```python
    import requests
    
    def get_all_issues(organization, state="open"):
        issues = []
        url = "https://nest.owasp.org/api/v0/issues/"
        params = {"organization": organization, "state": state}
        
        while url:
            response = requests.get(url, params=params)
            data = response.json()
            issues.extend(data["results"])
            url = data["next"]
            params = {}  # Next URL already includes params
        
        return issues
    
    all_issues = get_all_issues("OWASP")
    ```
  </Tab>
  
  <Tab title="JavaScript">
    ```javascript
    async function getAllIssues(organization, state = 'open') {
      let issues = [];
      let url = 'https://nest.owasp.org/api/v0/issues/';
      let params = { organization, state };
      
      while (url) {
        const response = await fetch(
          url + (params ? '?' + new URLSearchParams(params) : '')
        );
        const data = await response.json();
        issues.push(...data.results);
        url = data.next;
        params = null; // Next URL includes params
      }
      
      return issues;
    }
    
    const allIssues = await getAllIssues('OWASP');
    ```
  </Tab>
</Tabs>

## Best Practices for Issue Selection

<AccordionGroup>
  <Accordion title="Start Small">
    Begin with `good first issue` labels to understand:
    - Project structure and conventions
    - Development workflow
    - Testing requirements
    - Code review process
  </Accordion>
  
  <Accordion title="Check for Assignment">
    Before starting work:
    1. Read the issue completely
    2. Check if someone is already assigned
    3. Comment asking to be assigned
    4. Wait for maintainer confirmation
    
    <Warning>
      PRs for issues you're not assigned to will be automatically closed.
    </Warning>
  </Accordion>
  
  <Accordion title="Assess Time Commitment">
    Look for clues about issue complexity:
    - **Quick wins**: Documentation, typos, simple bug fixes
    - **Medium effort**: UI improvements, basic features
    - **Large effort**: Architecture changes, complex features
  </Accordion>
  
  <Accordion title="Understand Context">
    Before claiming an issue:
    1. Read all comments and discussion
    2. Check if there are linked PRs or related issues
    3. Review any design documents or specifications
    4. Ask questions if anything is unclear
  </Accordion>
</AccordionGroup>

## Common Issue Labels

| Label | Description | Experience Level |
|-------|-------------|------------------|
| `good first issue` | Beginner-friendly issue | Beginner |
| `bug` | Something isn't working | All levels |
| `enhancement` | New feature or request | Intermediate |
| `documentation` | Documentation improvements | Beginner |
| `help wanted` | Extra attention needed | All levels |
| `priority: high` | Urgent issue | Experienced |
| `wontfix` | Will not be worked on | N/A |
